Central Pharmacy & Outlet Pharmacy Management System (SRS)






1. Introduction
1.1 Purpose
This document defines the functional and business requirements for the Central Pharmacy & Outlet Pharmacy Management System. The module manages medicine procurement, central storage, distribution to outlet pharmacies, inter‑pharmacy transfers, sales, returns, stock adjustments, and accounting‑ready reporting.
1.2 Scope
The system supports:
One Central Store (main warehouse)
Multiple Outlet Pharmacies (OPD/IPD/Departmental)
Individual stock maintenance per pharmacy
Centralized approval and audit control
1.3 Definitions
Central Store: Main warehouse receiving medicines from suppliers
Outlet Pharmacy: Pharmacy unit dispensing medicines
Requisition: Request for medicine transfer
GRN: Goods Receive Note

2. System Overview
2.1 High-Level Process Flow
Supplier → Central Store (Receive)
Central Store → Outlet Pharmacy (Distribution)
Outlet ↔ Outlet (Inter‑pharmacy transfer)
Outlet → Patient / Department (Issue / Sale)
Returns, adjustments, and reporting

5.4 Central Pharmacy to Supplier Requisition (PO)

5.4.1. Purpose
The Central Pharmacy to Supplier Requisition (PO) module is designed to automate medicine procurement from suppliers based on consumption trends, current stock levels, and forecasted demand.
This module helps:
Maintain optimal inventory levels
Prevent stock-out situations
Reduce manual calculation errors
Improve procurement planning using data-driven decisions

5.4.2. Process Flow
Central Pharmacy → Create Requisition → Approval → Generate PO → Supplier
Step 1: Create Requisition
The pharmacy user initiates a purchase requisition based on sales history and stock availability.

5.4.5. Automated Calculation Logic
System calculates Request Quantity automatically using:
Step A — Calculate Average Sale
The application takes the day count from the date range input fields
Calculates the average sale.
Average Sale = Total Sale Quantity / Day count

Step B — Forecast Required Quantity
Forecast Required Quantity = Average Daily Sale × No of Days (Taken as input from user)

Step C — Request Quantity Calculation
If Forecast Required Quantity > Current Stock
Request Quantity = Forecast Required Quantity - Current Stock (View Only)
Order Quantity = Request Quantity (Editable by requester)
